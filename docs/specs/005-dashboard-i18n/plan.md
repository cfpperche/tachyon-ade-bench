# 005 — dashboard-i18n — plan

_Drafted from `spec.md` on 2026-07-07. The approach, not the steps (those go in `tasks.md`)._

## Approach

Add a small static i18n layer to the Astro dashboard. Pages declare their locale,
read shared UI copy from `src/lib/i18n.ts`, and pass localized labels into Astro
components and React islands. English remains the default route tree; Portuguese
routes are generated by mirrored pages under `src/pages/pt/`.

## Key decisions

- **Static `/pt/` routes** — chosen because GitHub Pages serves static output
  cleanly; rejected runtime locale negotiation because it adds state and edge
  behavior that the current dashboard does not need.
- **Shared UI dictionary** — chosen because labels should not drift across
  overview, matrix, profile, protocol, and source pages; rejected duplicating
  every literal in every localized page because it would make follow-up edits
  brittle.
- **Canonical competitor data remains untranslated** — chosen because the JSON
  files are source-backed research records; rejected duplicating data per locale
  until there is a translation review workflow for factual claims.

## Files touched

- `apps/bench-dashboard/src/lib/i18n.ts` — locale types, route helper, and UI
  dictionaries.
- `apps/bench-dashboard/src/lib/routes.ts` — locale-aware public URL helper.
- `apps/bench-dashboard/src/lib/format.ts` — localized labels for taxonomy values.
- `apps/bench-dashboard/src/layouts/BaseLayout.astro` — localized shell,
  navigation, and language switch.
- `apps/bench-dashboard/src/components/*` — localized badges, charts, protocol
  diagram, and matrix labels.
- `apps/bench-dashboard/src/pages/*` — English pages wired to locale-aware copy.
- `apps/bench-dashboard/src/pages/pt/*` — Portuguese static route tree.
- `apps/bench-dashboard/src/styles/global.css` — language switch styling.
- `README.md` — public English and Portuguese dashboard URLs.

## Risks & unknowns

- Route helpers can accidentally double-prefix `/pt/`; verify language-switch
  links on root, matrix, profile, protocol, and sources pages.
- React island props must stay serializable for Astro static builds.
- Portuguese UI strings can expand and break compact controls; inspect the
  rendered pages before publishing.

## Visual impact

Visible changes affect every dashboard page: language switch in the header,
Portuguese copy under `/pt/`, and localized labels inside filters, charts,
badges, and the protocol diagram. Capture browser evidence for English root,
Portuguese root, and Portuguese matrix/profile routing.
